# ЗАМІТКА: очищення git після переїзду на інший комп'ютер

> Цей файл — нагадування. На старті в git запушено **АБСОЛЮТНО ВСЕ** (включно з
> важкими референсними ассетами та дизайн-правилами), щоб проєкт можна було
> повністю клонувати й продовжити роботу на іншому комп'ютері.
>
> **Після того, як проєкт розгорнуто та робота продовжена на новому компі —
> прибери з git те, що не повинно там лежати.**

---

## Що варто прибрати з git (коли вже не потрібне для переїзду)

### 1. Референсний шаблон Webflow (найважче — ~основна частина 32M ассетів)
- `assets/saveweb2zip-com-repairly-webflow-io/` — повністю завантажений сторонній
  шаблон "Repairly". Потрібен лише як візуальний референс. У фінальному репозиторії
  готового сайту йому не місце.

### 2. Дизайн-«скіли» / правила (за бажанням)
- `.claude/rules/` — бібліотека дизайн-директив. Корисна під час розробки,
  але не є частиною готового сайту. Можна лишити для зручності або прибрати.
- `.sixth/` — наразі порожня папка.

### 3. Робочі/системні файли
- `.DS_Store` — вже у `.gitignore`, але якщо якісь потрапили в історію — прибрати.
- Сам цей файл `ЗАМІТКА-GIT.md` — видалити, коли стане непотрібним.

---

## Як саме прибрати (приклад команд)

Прибрати з відстеження git, але **лишити файли на диску** локально:

```bash
# приклад: прибрати важкий референсний шаблон з git, лишивши на диску
git rm -r --cached "assets/saveweb2zip-com-repairly-webflow-io"

# додати в .gitignore, щоб не повертався
echo "assets/saveweb2zip-com-repairly-webflow-io/" >> .gitignore

git commit -m "Прибрати референсний шаблон Webflow з git"
git push
```

Те саме для будь-якої іншої папки/файлу — поміняй шлях.

> Якщо треба **повністю стерти важкі файли з історії git** (щоб репозиторій став
> легшим), знадобиться `git filter-repo` або BFG Repo-Cleaner — це переписує
> історію, роби обережно і лише коли впевнений.
